# Known Bugs, Edge Cases & Mitigation Strategies

This document serves as the central tracker for known issues, edge cases, and systemic limitations within the current architecture. It defines the priority, reproduction steps, and planned mitigation strategies for each issue.

## 🔴 High Priority

### WebGL Context Loss on Mobile Safari
- **Description:** If a user on older iOS devices (or heavily resource-constrained mobile browsers) toggles between Recruiter Mode and WebGL mode rapidly, the browser's GPU context may fail to restore, resulting in a black canvas.
- **Reproduction:** Load site on iPhone 11 (Safari) -> Tap Recruiter Toggle 5+ times in rapid succession.
- **Root Cause:** iOS Safari strictly limits active WebGL contexts and aggressively dumps them if memory spikes.
- **Mitigation Strategy:** Implement a strict debounce on the `RecruiterToggle` component. Furthermore, ensure robust `dispose()` routines are called on `unmount` within `Scene.tsx` to explicitly clear Three.js geometries and materials from GPU memory.

## 🟡 Medium Priority

### Procedural Audio Glitch on Initial Interaction
- **Description:** Chrome and Safari's strict autoplay policies occasionally block the very first procedural audio click (generated by `audio.ts`) if the user hasn't sufficiently interacted with the DOM prior to the event.
- **Reproduction:** Open site in Incognito -> Wait 5 seconds -> Click immediately on a 3D mesh. Audio may fail to play, triggering a console warning.
- **Root Cause:** `AudioContext` is in a `suspended` state until a user gesture is registered on the document body.
- **Mitigation Strategy:** Attach a global, one-time `touchstart` or `click` listener on the `window` object that explicitly calls `audioCtx.resume()` before any component-level audio is requested.

## ⚠️ Systemic Limitations

### No SSR for 3D Assets
- **Description:** Because we utilize `window` and DOM-specific logic within React Three Fiber, the `<Scene />` canvas must be strictly rendered client-side using Astro's `client:only="react"` directive.
- **Impact:** There is a brief (usually < 100ms) hydration delay before the 3D scene appears.
- **Mitigation:** We mitigate this perceived latency by ensuring the CSS background color of the container perfectly matches the initial theme's background color, providing a seamless visual bridge while the GPU compiles the shaders.
